# ADR-001: RFC 7807 для ошибок API
Дата: 2025-09-22
Статус: Accepted

## Context
Нужно единообразно возвращать ошибки API, не раскрывая PII/внутренних деталей. Альтернативы: «как есть» (разные форматы), собственный формат, RFC7807.

## Decision
Используем RFC 7807 (application/problem+json): поля type, title, status, detail, correlation_id. Маппинг ошибок централизован. Логи связываем через correlation_id.

## Consequences
+ Единообразный контракт, проще тестировать и логировать.
− Небольшая обвязка/поддержка.
