# ADR-003: Источник секретов (JWT SECRET_KEY) и запрет в коде/логах
Дата: 2025-09-22
Статус: Accepted

## Context
Секреты не должны храниться в коде/репозитории. Альтернативы: .env в репо, переменные окружения, KMS/Vault.

## Decision
SECRET_KEY читается из окружения, не логируется. Ротация по регламенту.

## Consequences
+ Нет секретов в коде; совместимо с CI/CD.
− Нужно управлять окружениями.
